# Code Of Conduct

Analyze Creator Content is meant to make creator research more rigorous, respectful, and auditable. Contributors and participants are expected to:

- be constructive and specific,
- respect privacy, consent, platform boundaries, and content rights,
- use synthetic or clearly authorized data in issues and pull requests,
- avoid harassment, discriminatory language, impersonation, or targeted abuse,
- never post credentials, session material, private creator data, or working exploits.

Unacceptable behavior may be removed and repeat participants may be restricted from the project.

## Reporting

For an ordinary conduct concern that is safe to discuss publicly, use a GitHub issue. For sensitive reports, security issues, or reports containing private material, use GitHub private vulnerability reporting as described in [SECURITY.md](SECURITY.md). Never put personal or confidential details in a public issue. The maintainer will review reports in good faith, minimize unnecessary disclosure, and act proportionately.

By participating, you agree to follow this code in repository discussions, issues, pull requests, and other project spaces.
